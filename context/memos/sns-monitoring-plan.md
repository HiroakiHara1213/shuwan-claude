## 1. 背景・目的

SHUWANの市場監視を強化するため、SNS・動画プラットフォームからの情報収集体制を構築する。
目的は「顧客の購買動機・口コミ・競合動向・バズトレンド」をリアルタイムに把握すること。

## 2. MCP連携調査結果（2026-03-27実施）

検索キーワード: instagram, twitter, X, tiktok, youtube, facebook, social media, SNS, hootsuite, buffer, socialdog, sprout, social monitoring, analytics
→ すべて結果なし（0件）

## 3. 各SNSプラットフォームの技術的制約

| プラットフォーム | WebSearch | WebFetch | API | 制約・備考 |
|--------------|-----------|----------|-----|----------|
| **YouTube** | ○ | ○ | ○ Data API v3（無料） | 最も情報取得しやすい。動画検索・コメント・チャンネル情報すべて可 |
| **X (Twitter)** | ○ | △ | △ Basic $100/月 | WebSearchで間接取得可。API直接利用は有料 |
| **TikTok** | ○ | △ | × | WebSearchでバズ動画情報は取得可。動画内容の詳細分析は不可 |
| **Instagram** | △ | × | △ Meta Graph API（審査必要） | ログインウォール。公開投稿のハッシュタグ検索はWebSearchで部分的 |
| **Facebook** | △ | × | △ Meta Graph API（審査必要） | 公開ページ・グループの一部のみWebSearchで取得 |

## 5. 情報収集の仕組み（全体フロー）

```
[Google Alerts] → Gmail → 日次PDCA Gmail読み込み → context/market/ に記録
[YouTube MCP] → Researcher直接利用 → context/market/competitors.md / reviews-wom.md
[WebSearch] → Researcher日次チェック → context/market/ 各ファイルに記録
[Meta API] → （将来）Researcher直接利用 → context/market/reviews-wom.md
```

## 7. 見直し基準

- **月次見直し**: 各施策の情報収集量・品質を評価し、費用対効果を判断
- **追加検討トリガー**:
  - WebSearch間接取得で捕捉できない重要情報が発覚した場合
  - SNS上でSHUWAN言及が増加し、詳細分析が必要になった場合
  - 新しいMCPコネクタが公開された場合（MCPレジストリを定期確認）
  - 競合がSNS施策を強化し、詳細な競合分析が必要になった場合
