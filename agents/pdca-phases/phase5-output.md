# フェーズ5: アウトプット生成

最終的なアウトプットを `~/.claude/output/` 配下に生成する。

## 1. 日次レビューサマリー
`output/daily-review-YYYY-MM-DD.md`
- 今日のハイライト（3行以内）
- KPI進捗ダッシュボード（テーブル形式）
- 各エージェントの更新内容サマリー
- 明日のアクションアイテム

## 2. 変更ログ
`output/changelog.md` に追記
- どのファイルが更新されたか
- 何が変わったか（before/after）

## 3. Excelレポート再生成【必須・省略不可】

## 4. Notion書き込み【必須・省略不可】

日次レビュー結果をNotionデータベース「日次PDCAログ」に保存する。
データベースID: `33df339e-9adb-81a9-86d6-cfaa611c3ddf`

`API-post-page` で以下のプロパティを設定してレコードを作成：

| プロパティ | 値 |
|---|---|
| 名前 | `YYYY-MM-DD 日次レビュー` |
| 日付 | 当日の日付 |
| フェーズ | `Check` |
| ステータス | `完了`（全フェーズ完了時）/ `未完了`（途中で中断した場合） |
| KPI進捗 | 当月着地見込サマリー（チャネル別GAP要約、200文字以内） |
| チャンス・課題 | 優先度「高」の項目を箇条書き（200文字以内） |
| 活動修正 | 追加/強化/中止すべき活動の要約（200文字以内） |
| 要対応事項 | ユーザー判断待ちの事項（なければ「なし」） |

> **注意**: rich_textプロパティは2000文字制限があるため、要約して記録すること。詳細は `output/daily-review-YYYY-MM-DD.md` を参照。
