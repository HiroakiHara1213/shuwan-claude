# フェーズ3: 妥当性判断（CMOが統括）

フェーズ2の3エージェントの結果（+ フェーズ0のResearcher分析結果）を集約した後、@cmo を起動。

## @cmo 統括判断プロンプト

各エージェントのレビュー結果を統合し、以下を判断してください：

### 3-0. Notionログ横断分析（週次傾向把握）
- Notionデータベース（ID: `33df339e-9adb-81a9-86d6-cfaa611c3ddf`）から過去7日分のCheckレコードを取得
- `API-query-data-source` で日付降順・7件取得
- 過去7日のKPI進捗・チャンス/課題の推移からパターンを検出：
  - 同じチャネルが3日以上連続で未達 → 構造的問題として報告
  - 同じ課題が繰り返し出現 → 対応の優先度を引き上げ
  - 活動修正提案が実行されていない → 要対応事項として再掲
- レコードが不足している場合は「Notionログ蓄積中（N日分）」と明記して続行

### 3-0b. 月次切替チェック（月が替わった場合に自動実行）
- `monthly-kpi-tracking.md` の最新「当月（X月）着地見込」セクションのXと、今日の月（`new Date().getMonth()+1`）を比較
- **月が異なる場合**（例: ファイルが「3月」で今日が「4月」）は以下を実行：
  1. 新月セクション `### 当月（N月）着地見込 — YYYY-MM-DD v1` を `## 着地見込・GAP分析` の直後（旧当月セクションの前）に挿入
  2. 新月のKPI目標値は `monthly-kpi-tracking.md` の【A】テーブルから自動取得
  3. 実績・着地見込は「データ収集中」として記入し、PDCAで順次更新
  4. ダッシュボードは新月セクションを自動で表示する（パーサーが最初のセクションを読む）
- **月が同じ場合**: 既存セクションを最新データで上書き更新

### 3-1. 月次KPI進捗・着地見込作成（前月比付き）
- context/strategy/monthly-kpi-tracking.md を読み込み、当月のKPI目標を確認
- Researcher収集データ（market-trends.md内のアクセス解析・Shopifyファネル・セラーセントラル・Google広告実績）から当月の実績値を抽出
- 着地見込を算出: 月中実績 × (月間日数/経過日数) + パイプライン確度加重
- GAP = 着地見込 − 月次目標 を各チャネル別に算出
- **前月実績の抽出**: `monthly-kpi-tracking.md` の直前月セクション（例: 3月が当月なら2月セクション）から各チャネルの確定実績（碗数・利益）を抽出。前月セクションが存在しない場合は「情報不足」と明記
- **前月比の算出**: 前月比(%) = 当月着地見込 ÷ 前月実績 × 100（小数点第1位四捨五入）
- 判定: ◎120%以上 / ○100-119% / △80-99% / ×80%未満

### 3-2. GAP要因分析
- 未達見込チャネルの要因を特定（流入不足? CVR低下? 単価低下? パイプライン不足?）
- 超過見込チャネルの成功要因を特定（どの施策が効いているか）
- KPI逆算値（必要PV・必要購入人数・必要TOP流入）と実績の乖離を確認

### 3-3. チャンス・課題の特定
- Researcher分析（市場トレンド・競合動向・口コミ・検索トレンド・広告効率）から機会と脅威を抽出
- 各エージェント（Communication Planner・CSO・CTO）のレビュー結果から内部の強みと課題を抽出
- チャンス・課題を影響度（大/中/小）で優先順位付け

### 3-4. 活動修正提案
- 月次活動計画（②シート: context/memos/SHUWAN_月次KPI活動計画_v3 (18).xlsx の②月次活動計画）と照合
- 当月予定の活動で遅延・未着手のものを洗い出し
- GAP要因に基づき、追加すべき活動・強化すべき活動・中止すべき活動を提案
- Google広告（PMAX・リスティング）の検索語句分析から、広告戦略の修正提案

### 3-5. 戦略修正提案
- 年間目標1億円に対する現在の達成率と年間着地予測
- チャネル配分の見直し必要性（成長チャネルへのリソースシフト等）
- 四半期マイルストーン（kpi-targets.md）に対する進捗と修正案

### 3-6. 数値整合性チェック
各ファイル間の数値に矛盾がないか

### 3-7. FILE_MAP整合性チェック
各エージェントの定義（~/.claude/agents/*.md）のTodoWriteタスク項目と、ダッシュボードのFILE_MAP（~/ai-agent-dashboard/vite.config.ts）が一致しているか確認。ズレがあれば要対応事項として報告する。
